# 运行说明

本文档用于说明《中小型养老院信息管理系统的设计与实现》的本地运行步骤。

## 1. 环境准备

建议安装以下环境：

| 环境 | 推荐版本 | 说明 |
| --- | --- | --- |
| JDK | 17 | 后端项目运行环境 |
| Maven | 3.8+ | 后端依赖管理和启动 |
| Node.js | 18+ | 前端项目运行环境 |
| MySQL | 8.0+ | 数据库服务 |
| npm | Node.js 自带 | 前端依赖安装 |

可以通过以下命令检查环境：

```bash
java -version
mvn -version
node -v
npm -v
mysql --version
```

## 2. MySQL 数据库创建

项目数据库名称为 `nursing_home`。

在项目根目录执行建库建表脚本：

```bash
mysql --default-character-set=utf8mb4 -uroot -p123456 < database/nursing_home.sql
```

导入演示数据：

```bash
mysql --default-character-set=utf8mb4 -uroot -p123456 nursing_home < database/init.sql
```

如果已经进入 MySQL 命令行，可以执行：

```sql
source database/nursing_home.sql;
source database/init.sql;
```

导入后可以检查用户表：

```sql
USE nursing_home;
SELECT id, username, real_name, role, status FROM sys_user;
```

## 3. 后端配置和启动

后端配置文件：

```text
backend/src/main/resources/application.yml
```

当前默认数据库配置：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/nursing_home?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
```

如果本机 MySQL 密码不同，请先修改 `username` 和 `password`。

启动后端：

```bash
cd backend
mvn spring-boot:run
```

后端默认地址：

```text
http://localhost:8080
```

## 4. 前端安装依赖和启动

进入前端目录：

```bash
cd frontend
```

安装依赖：

```bash
npm install
```

启动前端：

```bash
npm run dev
```

前端默认地址：

```text
http://127.0.0.1:5173
```

前端请求 `/api` 会通过 Vite 代理转发到：

```text
http://localhost:8080
```

## 5. 登录测试

启动前后端后，打开前端地址并使用以下账号测试：

| 角色 | 用户名 | 密码 | 预期效果 |
| --- | --- | --- | --- |
| 管理员 | admin | 123456 | 显示全部管理菜单 |
| 员工 | staff01 | 123456 | 显示员工可访问菜单 |
| 家属 | family01 | 123456 | 显示家属用户中心和个人相关信息 |

也可以点击登录页的注册入口，注册新的家属账号。注册账号默认角色为 `FAMILY`。

## 6. 常见启动问题处理

### 后端端口被占用

后端默认端口是 `8080`。如果端口被占用，可以关闭占用进程，或修改 `application.yml` 中的端口配置。

### 数据库连接失败

处理思路：

- 检查 MySQL 是否启动。
- 检查数据库 `nursing_home` 是否存在。
- 检查账号密码是否和 `application.yml` 一致。
- 检查是否已经执行 `database/nursing_home.sql` 和 `database/init.sql`。

### 前端接口请求失败

处理思路：

- 确认后端已启动。
- 确认前端代理配置存在。
- 检查浏览器控制台的接口地址和错误信息。
- 检查后端控制台是否有接口异常。

### 前端依赖安装失败

处理思路：

- 确认 Node.js 版本建议为 18+。
- 重新执行 `npm install`。
- 网络较慢时可以更换 npm 镜像源。

### 登录失败

处理思路：

- 确认已经导入演示数据。
- 使用 `admin / 123456`、`staff01 / 123456`、`family01 / 123456` 测试。
- 检查数据库 `sys_user` 表中用户状态是否为“正常”。
