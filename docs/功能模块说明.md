# 功能模块说明

本文档按照当前项目实际功能整理各模块的作用、字段、页面、接口和答辩展示重点。

## 1. 登录注册模块

模块作用：

提供系统登录和家属用户注册入口，登录成功后返回用户基础信息和角色信息。

主要字段：

- 用户名
- 密码
- 确认密码
- 真实姓名
- 手机号
- 角色
- 状态
- 绑定老人 ID

主要功能：

- 用户登录
- 家属用户注册
- 登录后保存当前用户信息
- 退出登录并清除用户信息

对应页面：

- 登录页面
- 注册页面

对应接口：

- `POST /api/auth/login`
- `POST /api/auth/register`

答辩展示重点：

说明系统支持管理员、员工、家属三种角色，普通注册只能注册家属用户，避免普通用户注册为管理员或员工。

## 2. 用户角色模块

模块作用：

根据用户角色控制左侧菜单显示范围，并支持管理员维护系统用户账号。

主要字段：

- id
- username
- password
- real_name
- phone
- role
- status
- elderly_id
- create_time
- update_time

主要功能：

- 用户账号新增
- 用户账号删除
- 用户账号修改
- 用户账号查询
- 用户账号分页查询
- 不同角色菜单展示

对应页面：

- 用户账号管理页面
- 后台主布局左侧菜单

对应接口：

- `GET /api/sys-users/page`
- `GET /api/sys-users/{id}`
- `POST /api/sys-users`
- `PUT /api/sys-users/{id}`
- `DELETE /api/sys-users/{id}`

答辩展示重点：

使用管理员、员工、家属三个账号分别登录，对比不同角色看到的菜单差异。

- id
- name
- gender
- age
- phone
- id_card
- family_phone
- address
- check_in_date
- status
- remark
- create_time
- update_time

主要功能：

- 老人信息新增
- 老人信息删除
- 老人信息修改
- 老人信息查询
- 老人信息分页查询

对应页面：

- 老人信息管理页面
- 家属角色的我的老人信息页面

对应接口：

- `GET /api/elderly/page`

答辩展示重点：

说明老人信息是系统核心基础数据，后续健康、护理、缴费等模块都可以围绕老人展开。

## 10. 首页仪表盘模块

模块作用：

汇总展示系统关键运营数据，帮助管理员、员工和家属快速了解当前系统信息。

主要字段：

- elderlyCount
- staffCount
- totalBedCount
- occupiedBedCount
- freeBedCount
- healthAttentionCount
- unpaidCount
- noticeCount
- todayNursingCount

主要功能：

- 首页统计数据查询
- 统计卡片展示
- 运营概况展示
- 最近公告展示
- 不同角色首页标题展示

对应页面：

- 管理员后台管理中心
- 员工工作台
- 家属用户中心

对应接口：

- `GET /api/dashboard/summary`

答辩展示重点：

说明首页统计数据来自已有业务表，不是单纯静态页面，可以直观展示养老院当前运营概况。
