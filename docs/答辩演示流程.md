# 答辩演示流程

本文档提供一个 5-8 分钟的演示流程，语言尽量贴近毕业答辩现场，可以根据老师提问灵活调整。

## 1. 项目背景介绍

演示操作：

打开系统登录页面，先不登录，展示系统名称和入口。

讲解话术：

“老师好，我的毕业设计题目是《中小型养老院信息管理系统的设计与实现》。这个系统主要面向中小型养老院的日常管理场景，解决老人信息、员工信息、床位安排、健康档案、护理记录、缴费记录和公告通知分散管理的问题。系统采用前后端分离结构，目标是做出一个可以实际运行、功能完整、适合演示的后台管理系统。”

## 2. 技术架构介绍

演示操作：

可以切换到项目目录结构截图，或简单展示 README 中的技术栈。

讲解话术：

“系统后端使用 Spring Boot，数据访问使用 MyBatis-Plus，数据库使用 MySQL。前端使用 Vue 3、Vite 和 Element Plus，接口请求使用 Axios，页面路由使用 Vue Router，登录用户信息使用 Pinia 保存。前端通过 `/api` 代理访问后端接口，整体结构比较清晰，方便后续扩展。”

## 3. 登录和注册演示

演示操作：

1. 点击登录页的注册入口。
2. 展示家属注册表单。
3. 返回登录页。
4. 使用管理员账号 `admin / 123456` 登录。

讲解话术：

“系统支持多角色登录，包括管理员、员工和家属用户。普通注册入口只允许注册家属用户，不能注册管理员和员工，这样可以避免普通用户获得管理权限。管理员和员工账号一般由初始化数据或管理员创建。”

## 4. 管理员模式演示

演示操作：

管理员登录后展示首页仪表盘和左侧完整菜单。

讲解话术：

“现在进入的是管理员模式。管理员拥有系统全部功能，可以看到首页仪表盘、老人信息、员工信息、房间床位、健康档案、护理记录、缴费管理、公告管理和用户账号管理。首页通过卡片展示老人数量、员工数量、床位数量、未缴费记录和今日护理记录等关键数据，方便管理人员快速了解养老院当前运营情况。”

## 5. 员工模式演示

演示操作：

退出登录，使用 `staff01 / 123456` 登录，展示员工菜单。

讲解话术：

“这是员工角色登录后的页面。员工主要负责日常护理和业务记录，所以系统只展示老人信息查看、房间床位查看、健康档案管理、护理记录管理和公告查看等功能。像员工信息管理、缴费管理、用户账号管理这些模块不会显示给员工。”

## 6. 家属用户模式演示

演示操作：

退出登录，使用 `family01 / 123456` 登录，展示家属菜单。

讲解话术：

“这是家属用户中心。家属用户主要用于查看自己绑定老人的相关信息，比如我的老人信息、我的健康档案、我的护理记录、我的缴费记录和公告查看。家属角色默认以查看为主，不提供新增、编辑、删除业务数据的入口，适合家属了解老人近期情况。”

## 7. 核心业务模块演示

演示操作：

切回管理员账号，依次打开几个核心页面：

- 老人信息管理
- 房间床位管理
- 健康档案管理
- 护理记录管理
- 缴费管理
- 公告管理

可以选择一个模块演示新增、编辑、查询、分页和删除确认。

讲解话术：

“核心业务模块都采用统一的列表管理方式，包括搜索栏、表格、分页、新增按钮、编辑弹窗和删除确认。老人信息是基础数据，健康档案和护理记录可以关联老人，护理记录还可以关联护理员工。房间床位模块用于管理床位状态，缴费模块用于记录不同费用类型和缴费状态，公告模块用于发布系统公告和活动通知。”

## 8. 首页统计数据演示

演示操作：

回到首页，展示统计卡片、运营概况和最近公告。

讲解话术：

“首页统计数据来自已有业务表，例如老人信息表、员工信息表、房间床位表、健康档案表、护理记录表、缴费记录表和公告表。这里不是写死的静态页面，而是后端提供汇总接口，前端展示统计结果。这样管理员可以在首页直接看到系统运行情况。”

## 9. 项目总结

演示操作：

停留在首页或项目功能模块页面。

讲解话术：

“整体来说，本系统完成了养老院日常管理中比较核心的功能，包括基础信息维护、床位管理、健康和护理记录、缴费记录、公告通知以及多角色登录和菜单控制。系统采用前后端分离开发，界面风格统一，数据可以通过 MySQL 初始化脚本快速导入，能够满足毕业设计运行和答辩演示要求。后续如果继续扩展，可以加入更细的权限控制、数据导出、图表统计和消息提醒等功能。”
